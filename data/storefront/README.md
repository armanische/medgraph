# Storefront catalog

`data/storefront` — Git-backed источник данных будущей публичной витрины
CyberMedica. Текущие публичные страницы пока не используют этот каталог.

## Структура

- `products/*.json` — один товар в одном файле;
- `manufacturers/*.json` — один производитель в одном файле;
- `categories/*.json` — одна категория в одном файле;
- `catalog-summary.json` — версия схемы и контрольные количества сущностей.

Имя файла должно совпадать со `slug` сущности. JSON проверяется строгими Zod-схемами
из `lib/storefront/schemas.ts`; неизвестные поля отклоняются.

## Как добавить товар

1. Создайте или выберите активного производителя и категорию.
2. Создайте `products/<slug>.json` со всеми полями модели `Product`.
3. Используйте стабильный `id`; не переиспользуйте ID удалённых товаров.
4. Обновите `catalog-summary.json`.
5. Запустите storefront tests, TypeScript и build.

Все `id` и `slug` должны быть уникальными. `manufacturerId`, `categoryId`,
`relatedProductIds` и непустой `compatibleProductId` обязаны ссылаться на
существующие сущности каталога.

## Статусы

Товар:

- `active` — доступен в обычной публичной выдаче;
- `on_request` — доступен по запросу коммерческого предложения;
- `discontinued` — снят с предложения и исключён из активной выдачи;
- `hidden` — полностью скрыт из публичной выдачи.

Производитель и категория поддерживают только `active` и `hidden`.

Статусы Research, Review, Evidence, Verification и Publication запрещены. В JSON
нельзя добавлять evidence IDs, review IDs, artifact paths, SHA, reviewer notes,
publication history и другие внутренние поля. Строгая схема отклонит такие поля.

## Изображения и документы

- Не копируйте крупные файлы в Git без необходимости.
- Предпочитайте HTTPS-ссылки на разрешённые публичные ресурсы или object storage/CDN.
- Локальный путь допустим только как root-relative URL внутри `public/`.
- Перед публикацией проверяйте права на изображение или документ.
- `isOfficial: true` допустим только для документа официального источника.
- В каталоге хранится публичный URL, а не путь к research artifact.

## Будущая миграция

Публичные компоненты должны работать через `CatalogRepository`, а не читать JSON
напрямую. При переходе на CMS или Supabase нужно реализовать новый адаптер того же
интерфейса и заменить composition root в `lib/storefront/index.ts`. Модели и
публичные компоненты при этом не меняются.
